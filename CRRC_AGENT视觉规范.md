# 中国中车 CRRC 品牌色彩视觉规范（Agent 开发版）

> 来源：用户提供的《中国中车 VI 手册（第一版 jpg）》基础部分 A4 品牌色彩规范页面。本文面向 Agent / UI / 前端开发使用，提炼为可执行的色彩、搭配与标识应用规则。

## 1. 使用目标

在为中国中车（CRRC）相关系统、网页、看板、应用界面或生成式 Agent 输出物进行视觉设计时，应保持品牌色彩的一致性、识别性和庄重感。

核心原则：

- 品牌主色以“中车红”为最高识别色，用于强调、品牌露出、关键状态与核心视觉锚点。
- 黑、白、灰构成主要信息与版面骨架，保持专业、稳重、工业科技感。
- 金、银及特殊色只作为辅助或场景化补充，不应替代中车红的主品牌地位。
- 标识颜色必须根据背景明度和色阶选择，不得随意改色、降低对比度或使用禁用组合。

## 2. 色彩模式术语

| 色彩模式 | 使用场景 | 开发/设计说明 |
|---|---|---|
| PANTONE 专色 | 高品质或特殊印刷工艺 | 用于印刷色彩校准，是客户与印厂之间统一颜色的标准。 |
| CMYK 四色 | 印刷、包装、广告用品 | 用于印刷物料，需按 CMYK 数值输出。 |
| RGB 视屏色 | 屏幕、PPT、视频、数字界面 | 用于 Web、App、数据看板、Agent UI。 |
| HEX 网络安全色 | Web、跨设备浏览器显示 | Agent 生成网页/CSS 时优先使用 HEX。 |

## 5. 品牌标识色彩应用规则

### 5.1 单独图形标识

中车红背景色阶：

| 背景色阶 | 标识建议 |
|---|---|
| 51%-100% 中车红深色区 | 使用反白标识。 |
| 0%-50% 中车红浅色区 | 使用标准色中车红标识。 |

典雅黑背景色阶：

| 背景色阶 | 标识建议 |
|---|---|
| 0%-50% 黑色色阶 | 使用标准色标识。 |
| 51%-80% 黑色色阶 | 使用反白标识。 |
| 80% 以上黑色色阶 | 可使用标准色或反白标识，优先标准色标识。 |

### 5.2 标识组合（图形 + CRRC）

中车红背景色阶：

| 背景色阶 | 标识组合建议 |
|---|---|
| 51%-100% 中车红深色区 | 使用反白品牌标识组合。 |
| 0%-50% 中车红浅色区 | 使用标准色品牌标识组合。 |

典雅黑背景色阶：

| 背景色阶 | 标识组合建议 |
|---|---|
| 0%-50% 黑色色阶 | 使用标准色品牌标识组合。 |
| 51%-80% 黑色色阶 | 使用反白品牌标识组合。 |
| 80% 以上黑色色阶 | 可使用反白或半反白组合，优先半反白组合。 |

辅助灰色背景色阶：

| 背景色 | 色阶范围 | 标识建议 |
|---|---|---|
| 深墨色 | 0%-50% | 使用标准色标识/组合。 |
| 深墨色 | 51%-90% | 使用反白标识/组合。 |
| 深墨色 | 90% 以上 | 可使用反白或半反白组合，优先半反白组合。 |
| 浅墨色 | 0%-70% | 使用标准色标识/组合。 |
| 浅墨色 | 70% 以上 | 使用反白标识/组合。 |
| 淡墨色 | 全部色阶 | 使用标准色标识/组合。 |

### 5.3 色彩搭配专用表规则

手册给出不同底色下的“允许/禁止”关系，可归纳为：

- 红色底上不得再使用红稿标识，应使用黑稿、反白、金稿或银稿中对比足够的版本。
- 黑色底上不得使用黑稿标识，应优先使用反白或金银稿。
- 深灰、浅灰底上慎用低对比金银稿，避免标识消失。
- 金色底上禁止使用金稿标识，优先使用黑稿或反白稿。
- 银色底上禁止使用银稿标识，优先使用黑稿、红稿或反白稿。
- 当输出设备需要 RGB，但手册只给出印刷色时，应以印刷色值为准选择最接近的屏幕色。

## 6. 品牌色彩占比

手册以中国传统书画、水墨色彩为灵感，强调白色、浅墨、深墨占大部分比例，中车红起点睛作用。

推荐数字界面配色占比：

| 方案 | 白色 | 深墨/浅墨/黑 | 中车红 | 适用场景 |
|---|---:|---:|---:|---|
| 标准稳重型 | 50% | 43% | 7% | 企业官网、介绍页、品牌专题。 |
| 均衡信息型 | 50% | 40% | 10% | 管理后台、数据看板、业务系统。 |
| 极简强调型 | 80% | 0%-10% | 20% | 活动海报、重点营销页、品牌强曝光。 |
| 浅灰专业型 | 60% | 30% 浅墨 | 10% | 应用界面、仪表盘、文档型页面。 |

开发建议：

- 页面背景以白色或极浅灰为主，避免大面积红色造成视觉压迫。
- 中车红的界面占比建议控制在 7%-10%，只在品牌海报、启动页、专题页中放大至 20%。
- 文字主体应使用黑色或深墨色，辅助说明可用浅墨色。
- 数据可视化中不要把红色分配给大量普通分类；红色应保留给重点、异常、核心指标或品牌元素。

## 7. UI 开发落地建议

### 7.0 核心原则：多色搭配，红色点睛

> **重要：** 中车红 `#C70019` 是品牌识别色，**不是** UI 的唯一主色调。
>
> 正确做法：为系统选择一个适合人机交互的 UI 主色（如蓝色、靛蓝、深灰），用于导航、边框高亮、焦点状态、次级按钮等大量出现的交互元素；仅在少数关键节点引入中车红作为点缀与强调。
>
> 错误做法：把中车红设置为全局 `--accent` 变量，导致所有按钮、输入框边框、图标、高亮均呈红色——视觉疲劳，品牌感反而稀释。

### 7.1 色彩层级分配

| 层级 | 角色 | 推荐色 | 说明 |
|---|---|---|---|
| 背景骨架 | 页面主背景、容器 | 白 `#FFFFFF`、浅灰 `#F8FAFC`、深灰 `#0f1117` | 占 50%–60% |
| UI 主色 | 导航激活、边框高亮、焦点环、hover 态、一般标签 | 蓝 / 靛蓝 / 科技蓝 `#00678F` / 深灰 | 占 30%–40% |
| 中车红点缀 | 主操作按钮、品牌 Logo/标识、关键指标、重点告警 | `#C70019` | 占 5%–10% |
| 辅助功能色 | 成功 `#10b981`、警告 `#f59e0b`、错误 `#ef4444` | 保留语义色 | 不与中车红混用 |

### 7.2 中车红的正确使用场景

**应使用（点睛）：**
- 页面/应用的品牌标识图形（Logo、图标背景渐变）
- 最主要的"提交/确认"按钮（每屏通常只有 1 个）
- 关键数据指标的高亮数字、状态标记
- 特殊活动、强调型标题（用量极少）

**不应使用（避免滥用）：**
- 所有 hover 边框、focus 外发光（改用 UI 主色）
- 导航激活状态（改用 UI 主色）
- 标签、芯片、角标等大量重复出现的元素
- 辅助图标着色（保持灰色或 UI 主色）
- 数据图表的普通系列（仅用于最重要的那条线/指标）

### 7.3 页面布局

- 使用大面积留白，保持克制、稳定、国企品牌的秩序感。
- 主要内容容器使用白色或极浅灰背景，分隔线使用浅灰或 10%–20% 黑。
- 避免过度渐变、霓虹、高饱和多彩背景。
- 图像素材宜选择轨道交通、制造、科技、城市交通、工程场景，并可用灰度或低饱和处理。

### 7.4 按钮与交互状态

| 组件 | 建议 |
|---|---|
| 主按钮（每屏唯一） | 背景 `#C70019`，文字白色。 |
| 次按钮、一般操作 | 白底或浅灰底，文字深墨色，边框 UI 主色或浅墨色。 |
| 焦点环 / hover 边框 | 使用 UI 主色（蓝/靛蓝），不使用中车红。 |
| 危险操作 | 不复用品牌红；通过文案、图标和确认机制区分，如需标色可用 `#ef4444`。 |
| Hover | 主按钮加深红 8%–12%；次按钮用 UI 主色高亮。 |
| Disabled | 使用 `#B4B5B5` 或浅灰透明态，不使用品牌红。 |

### 7.5 数据可视化

- 品牌主线、目标线、最关键异常可使用 `#C70019`；一屏中最多 1–2 个系列使用。
- 普通系列使用灰阶、科技蓝 `#00678F`、环保绿 `#92B346` 等辅助色。
- 金色和黄色不要用于普通警告，以免与品牌特殊工艺色混淆。
- 在深色图表背景上优先使用反白文字，并保证对比度。

### 7.6 文字对比度（可读性）

- 正文文字须满足 WCAG AA 对比度（4.5:1）。
- 深色背景下辅助文字最深不应低于 `#64748b`，暗提示文字不低于 `#7c8a9e`。
- 浅色背景下辅助文字不应淡于 `#64748b`，暗提示文字不淡于 `#94a3b8`。
- 禁止低对比组合：浅灰底银色字、金色底金色标识、黑底黑标识、红底红标识。
- 重要信息不只依赖颜色表达，还应配合图标、标签或文字说明。

## 8. Agent 生成规范提示词片段

可在开发 Agent 中加入以下约束：

```text
你正在为中国中车 CRRC 品牌生成界面。

色彩策略：
- 页面骨架：白色、极浅灰（亮色模式）或深灰黑（暗色模式）作为主背景，占 50%–60%。
- UI 主色：选用蓝色、靛蓝或科技蓝 #00678F 作为导航激活、边框高亮、焦点状态、hover 效果、一般标签的颜色，占 30%–40%。
- 中车红点睛：仅将 #C70019 用于品牌 Logo 图形、每屏唯一的主操作按钮、核心关键指标高亮，占 5%–10%，绝对不要把红色作为全局 accent 覆盖所有交互元素。
- 辅助语义色：成功绿 #10b981、警告黄 #f59e0b、错误红 #ef4444 保留语义用途，不与中车红混用。

禁止：
- 将中车红设为全局 --accent，导致边框、焦点环、hover、标签全部变红。
- 花哨渐变、过度装饰、高饱和多彩背景。
- 品牌红在数据图表中用于普通分类系列。

Logo/标识在深色或红色背景上使用反白版本，在浅色背景上使用标准色版本，禁止低对比或同色标识。
```

## 9. Agent 开发检查清单

- [ ] 是否使用 `#C70019` 作为唯一品牌主红。
- [ ] 是否避免了红色大面积滥用。
- [ ] 标识在当前背景上是否有足够对比度。
- [ ] 是否遵守“深底反白、浅底标准色”的基本原则。
- [ ] 是否避免红底红标、黑底黑标、金底金标、银底银标。
- [ ] UI 背景是否以白色、灰阶为主。
- [ ] 主按钮、关键指标是否使用品牌红而非随机红。
- [ ] 数据图表中红色是否只用于关键含义。
- [ ] 特殊蓝、黄、绿是否仅用于对应业务/功能场景。
- [ ] 页面整体是否呈现稳重、清晰、工业科技感。
